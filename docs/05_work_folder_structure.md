# 05_作業フォルダ構成（正）
## 1. 目的
ETC2.0プローブデータ分析の作業を、案件ごとに再現可能・整理可能にするための標準フォルダ構成を定義する。重い共通データを分離し、案件単位の成果物を一貫した場所へ出力する。
## 2. フォルダ構成（正）
ETCプローブデータ分析（作業フォルダ）/
  00_共通データ/
    00_受領データ/                 # 受領データ（生データ・原本、書き換え禁止）
    01_第１スクリーニング/         # 第1スクリーニング結果（大量・月別）
    02_ゾーニングデータ/           # ゾーン定義（12_polygon_builder.html成果）
    03_背景地図・道路網/           # 任意：OSM抽出、道路中心線、背景タイル等（再利用）
    04_補助データ/                 # 任意：祝日、車種コード表、辞書、設定テンプレ等
  10_抽出データ/
    {PROJECT_ID}/                  # 例：YYYYMMDD_HHMM_2nd_point_{CROSSSET_NAME}_{YYYYMM}
      00_src/
        scripts/                   # プロジェクト固有に書き換えたスクリプト
        logs/                      # 実行ログ・作業ログ
        config/                    # 設定（paths.yaml等）、実行バッチ等
      10_ルート(Route)データ/       # 10_route_sampler.py の成果
      11_交差点(Point)データ/       # 11_crossroad_sampler.py の成果
      20_第２スクリーニング/        # 第２スクリーニングデータ
      30_ルートパフォーマンス/      # 30_build_performance.py の成果
      31_交差点パフォーマンス/      # 31_crossroad_trip_performance.py の成果
      40_様式1-3参照ODリスト/       # 40_trip_od_screening.py の成果
      41_ODヒートマップ/            # 41_od_heatmap_viewer.py の成果
      42_ODマトリクス/              # 42_OD_extractor.py の成果
      50_経路分析/                  # 50_Path_Analysis.py の成果
      80_中間成果/                  # 任意：検証・試作出力の退避先
      90_最終成果/                  # 報告書・図表・スライド等
      01_プロジェクト概要.txt       # 案件の目的・対象期間・条件・メモ
## 3. 運用ルール（重要）
- 00_受領データは原本扱いとし、書き換えない（必要なら案件側へコピーして加工する）。
- 全てのフォルダを事前に作成しない。作業して成果ができたもののみフォルダを作成する。
- ただし {PROJECT_ID}/00_src/config/ は再現性のため最初に用意する（推奨）。
- 出力先は「案件フォルダ配下の対応する番号フォルダ」に固定する。
## 4. PROJECT_ID 命名規則（推奨）
- 例：20260107_0930_2nd_point_TsuyamaST_202512
- 先頭に日時、次に処理種別、最後に対象セット名と対象年月を付け、一覧で時系列ソートできるようにする。
