# 10_route_sampler（ルートファイル作成ツール）

## 目的（何をする）
`10_UI_route_sampler.py`（PyQt6）と `10_route_sampler.html`（Leaflet + JavaScript）は、地図上でクリックしたルート点から後続処理用のルートCSVを作成するツールです。

交差点ファイル作成ツール `11_UI_crossroad_sampler.py` と同じ操作感・画面構成にしており、左側に作成済みルート一覧、右側にSTEPカードと地図を表示します。

## 位置づけ（分析フロー上のどこ）
- **ルート定義作成**フェーズ。
- PDF 用語での「ルート定義作成」に対応し、20系の第2スクリーニングや30系のルートパフォーマンスで参照する基準ルートを提供します。

## 入力
- プロジェクトフォルダ。
- 地図上でクリックしたルート点列。
- リサンプリングピッチ[m]。初期値は `20m` です。

## 出力
- 出力先: `{PROJECT_ID}/10_ルート(Route)データ/`。
- 出力CSV: ルート名の末尾にピッチを付けた `<route_name>_<pitch>m.csv`。
  - 例: `中心市街地ルート_10m.csv`
  - 小数ピッチはファイル名に使いやすいように `12p5m` のように表します。
- CSVはヘッダなしで、後続ツールが読む列互換を維持します。
  - O列 = 経度
  - P列 = 緯度
  - G列 = ダミーGPS時刻
  - M列 = FLAG `2`
  - S列 = ダミー速度 `30.0`

## 実行方法
- UI版: `python src/10_UI_route_sampler.py`
- embedded配布環境では `bat/10_route_sampler.bat` から起動します。
- 旧互換の簡易Flask版は `src/unreleased/10_route_sampler.py` に残していますが、通常運用では新しいPyQt6 UIを使います。

## UI操作
1. **STEP1 プロジェクトフォルダ選択**
   - 選択したプロジェクト配下に `10_ルート(Route)データ` を自動作成します。
2. **STEP2 地図でルート指定**
   - 左クリックでルート点を追加します。
   - 右クリックで直前の点を削除します。
   - クリアで現在編集中のルートを消去します。
3. **STEP3 ルートファイルの保存**
   - ルート名とピッチ[m]を指定します。
   - 保存時に指定ピッチ + 30°以上の折れ点保持で再サンプリングします。
   - ファイル名は自動で `ルート名_10m.csv` のようになります。
4. **STEP4 一括で次のルートへ**
   - 保存後は地図と入力欄をクリアし、続けて次のルートを作成できます。
   - 左側一覧から編集、リネーム、削除もできます。

## 判定ロジック（重要なものだけ）
- クリックした原始ルート点列を、指定ピッチ[m]で再サンプリングします。
- 30°以上の折れ点は必ず保持します。
- 保存ファイル名にはピッチを付与し、後からどの間隔で作ったルートか分かるようにします。
- 編集時は既存CSVのO/P列を読み戻し、地図上にルートとして再表示します。

## できること / できないこと
- できること: 1つのプロジェクトで複数ルートを連続作成し、ルート名・ピッチ付きCSVとして管理できます。
- できないこと: 実測走行データへの自動スナップ、道路ネットワークに沿った自動補正、オフライン地図タイルの保証。

## よくあるミス
- ルート点が1点だけで保存しようとする。
- ルート名にWindowsで使えない記号を入れる。保存時に `_` に置換されます。
- ピッチを大きくしすぎて、後続のKP集計が粗くなる。
