# ETC2.0 Analysis Toolkit (ETC)

ETC2.0 Analysis Toolkit は、  
ETC2.0プローブデータを行政の意思決定に活かすための  
**透明性重視・段階的解析型ツール群**です。

本ツールは、巨大な交通データをブラックボックス化せず、  
誰が見ても説明可能な形で処理し、  
可視化しながら検証できることを理念としています。

---

## なぜこのツールが必要か

ETC2.0データは強力です。  
しかし同時に、

- 処理過程が見えない
- 分析ロジックが属人化する
- 結果だけが独り歩きする

というリスクを抱えています。

本ツール群は、それらを回避するために設計されています。

---

## 基本思想

### 1. 段階的処理
データを一括で結論に導かない。  
前処理 → 抽出 → 確認 → 集計 → 可視化  
という工程を分離し、各段階で確認できる構造とする。

### 2. ブラックボックス化しない
ロジックは公開・明示。  
処理順は番号で管理し、再現性を確保する。

### 3. 可視化を前提とする
数値だけで判断しない。  
地図表示・ヒートマップ・集計表など、  
視覚的に確認できることを前提とする。

### 4. EBPMへの適合
行政の説明責任に耐える形式で整理する。  
「なぜこの結論に至ったか」を追跡可能にする。

### 5. 個人特定を目的としない
本ツールは交通流分析のためのものであり、  
個別車両や個人の特定を目的としない。

---

## このツール群で実現すること

- 巨大なプローブデータの段階的整理
- 必要なデータのみの抽出
- 交差点・路線・ゾーン単位での分析
- 流入・流出・OD・経路の可視化
- 分析過程の再現性確保

---

## リポジトリ全体に込められた思い

このリポジトリを見渡すと、単なる「解析コード集」ではなく、
**現場が引き継げる分析文化をつくる**という意思が一貫しています。

- `src/` と `docs/` が同じ番号体系で対応し、手順と実装を切り離さず運用する設計
- 第1/第2スクリーニング、OD、経路分析までを「一気通貫」ではなく段階ごとに検証可能にする設計
- GUI・ブラウザ・バッチを併用し、専門職だけでなく実務担当者にも扱える導線を残す配慮
- フォルダ命名や成果物配置まで明文化し、再解析や監査時に追跡しやすくする規律
- 「できること」と「できないこと」を明記し、分析結果の過信を防ぐ倫理的な姿勢

つまり ETC は、

> 精度だけでなく、説明可能性・継続可能性・責任ある運用を同時に満たす

ことを目指した実装基盤です。

---

## このツール群が目指す姿

交通政策は、  
「勘」や「経験」だけで語られるべきではありません。

しかし同時に、  
数値だけの無機質な分析でもあってはなりません。

本ツール群は、

> データを見える形で扱い、  
> 透明性を担保しながら、  
> 行政判断を支えるための基盤

を目指しています。

---

## 守秘と責任

本ツールは行政EBPM向けの分析補助を目的とします。

- 個人情報保護を遵守すること
- 契約上の守秘義務を守ること
- 分析結果を絶対値と誤認しないこと

最終的な政策判断の責任は、道路管理者にあります。

---

## 最後に

ETC2.0 Analysis Toolkit は  
単なる解析スクリプト群ではありません。

それは、

「交通データを、透明に、段階的に、説明可能に扱う」

という思想の実装です。
