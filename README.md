# ETC2.0 Analysis Toolkit (ETC)

ETC2.0 Analysis Toolkit は、  
ETC2.0プローブデータを行政の意思決定に活かすための  
**透明性重視・段階的解析型ツール群**です。

本ツールは、巨大な交通データをブラックボックス化せず、  
誰が見ても説明可能な形で処理し、  
可視化しながら検証できることを理念としています。

---

## なぜこのツールが必要か

ETC2.0データは強力です。  
しかし同時に、

- 処理過程が見えない
- 分析ロジックが属人化する
- 結果だけが独り歩きする

というリスクを抱えています。

本ツール群は、それらを回避するために設計されています。

---

## 基本思想

### 1. 段階的処理
データを一括で結論に導かない。  
前処理 → 抽出 → 確認 → 集計 → 可視化  
という工程を分離し、各段階で確認できる構造とする。

### 2. ブラックボックス化しない
ロジックは公開・明示。  
処理順は番号で管理し、再現性を確保する。

### 3. 可視化を前提とする
数値だけで判断しない。  
地図表示・ヒートマップ・集計表など、  
視覚的に確認できることを前提とする。

### 4. EBPMへの適合
行政の説明責任に耐える形式で整理する。  
「なぜこの結論に至ったか」を追跡可能にする。

### 5. 個人特定を目的としない
本ツールは交通流分析のためのものであり、  
個別車両や個人の特定を目的としない。

---

## このツール群で実現すること

- 巨大なプローブデータの段階的整理
- 必要なデータのみの抽出
- 交差点・路線・ゾーン単位での分析
- 流入・流出・OD・経路の可視化
- 分析過程の再現性確保

## ソフト群一覧

本ツール群は、用途ごとに複数のソフトで構成されています。  
主な構成は次のとおりです。

### 01　第1スクリーニング
- 第1スクリーニングUI
- 運行ID単位の分割・時系列並べ替え処理

### 02　時間帯存在トリップ集計＋ピーク30分OD抽出
- 指定日の30分帯ごとの存在トリップ集計
- ピーク30分帯の抽出
- 指定30分帯内の最初の点をO、最後の点をDとしたOD抽出 

### 03　推定拠点ゾーン対応
- 運行IDごとの推定拠点ゾーン対応表作成

### 05　トリップビューアー
- 抽出済みトリップの地図確認
- 個別トリップの挙動確認

### 11　交差点ファイル作成ツール
- 交差点中心と枝方向の定義
- 交差点CSV・JPGの作成  
- PyQtの管理UIとHTML地図画面で構成 

### 12　ポリゴンビルダー
- ゾーニング用ポリゴンの作成

### 21　第2スクリーニング
- 指定交差点を通過するトリップの抽出
- 第2スクリーニング済みCSVの作成

### 31　交差点パフォーマンス分析
- 流入枝番・流出枝番の判定
- 所要時間・遅れ時間の算出
- performance.csv の作成

### 32　交差点レポート作成
- performance.csv をもとにした Excel レポート自動生成

### 33　IN-OUT枝判定ビューアー
- 交差点パフォーマンス分析結果の妥当性確認
- 1トリップごとの枝判定結果を地図上で可視化

## リポジトリ全体に込められた思い

このリポジトリを見渡すと、単なる「解析コード集」ではなく、  
**現場が引き継げる分析文化をつくる**という意思が一貫しています。

- `src/` と `docs/` が同じ番号体系で対応し、手順と実装を切り離さず運用する設計
- 第1/第2スクリーニング、OD、経路分析までを「一気通貫」ではなく段階ごとに検証可能にする設計
- GUI・ブラウザ・バッチを併用し、専門職だけでなく実務担当者にも扱える導線を残す配慮
- フォルダ命名や成果物配置まで明文化し、再解析や監査時に追跡しやすくする規律
- 「できること」と「できないこと」を明記し、分析結果の過信を防ぐ倫理的な姿勢

つまり ETC は、

> 精度だけでなく、説明可能性・継続可能性・責任ある運用を同時に満たす

ことを目指した実装基盤です。

## 導入方法とフォルダ準備

本ツールは、一般的なソフトウェアのような**インストール作業は不要**です。  
必要なフォルダを所定の場所に配置し、バッチファイルから起動して利用します。

### ①「_EtcAnalyzer」フォルダ

ダウンロードしたZIPファイルには、分析ツール一式をまとめたフォルダ **「_EtcAnalyzer」** が格納されています。  
まず、このフォルダを **Cドライブ直下** または **任意のドライブ直下** に解凍してください。

例

C:\_EtcAnalyzer

#### bat
このフォルダには、プログラムを起動するための**バッチファイル**が格納されています。  
各ツールを起動する場合は、この中の `〇〇〇〇.bat` というファイルをダブルクリックしてください。

#### runtime
このフォルダには、Pythonプログラムを動かすために必要な実行環境が格納されています。  
**この中は触らないでください。**

#### src
このフォルダには、Pythonで記述されたプログラム本体が格納されています。  
**この中も触らないでください。**

---

### ②「_EtcData」フォルダ

次に、分析作業用のフォルダとして **「_EtcData」** というフォルダを**手動で新規作成**します。  
このフォルダには、解析時に中間データや結果データが大量に作成されます。  
そのため、**内蔵SSDなど、高速で十分な空き容量がある場所**に作成してください。

例

C:\_EtcData

目安として、**500GB程度の空き容量**がある場所を推奨します。  
さらに、あらかじめこの中に、必要に応じた作業用フォルダを**手動で作成**しておくと、分析を整理しやすくなります。

---

### ③ ETC2.0プローブデータの生データ

ETC2.0プローブデータ（様式1-2）の生データは、  
**必ずしも `_EtcData` フォルダ内に置く必要はありません。**

生データは非常に大容量となるため、むやみに移動させるのではなく、  
**外付けSSDや外付けHDDに保存したまま利用することも可能**です。

---

### 準備完了

以上で、分析ツールを利用するための準備は完了です。

通常のソフトウェアのように、特別なインストール作業は不要です。  
必要なフォルダを配置し、各バッチファイルから起動して利用してください。

## このツール群が目指す姿

交通政策は、  
「勘」や「経験」だけで語られるべきではありません。

しかし同時に、  
数値だけの無機質な分析でもあってはなりません。

本ツール群は、

> データを見える形で扱い、  
> 透明性を担保しながら、  
> 行政判断を支えるための基盤

を目指しています。
