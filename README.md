# ETC2.0アナライザー Toolkit (ETC)

ETC2.0アナライザー Toolkit は、  
ETC2.0プローブデータを行政の意思決定に活かすための  
**透明性重視・段階的解析型ツール群**です。

本ツールは、巨大な交通データをブラックボックス化せず、  
誰が見ても説明可能な形で処理し、  
可視化しながら検証できることを理念としています。

---

## 基本思想

### 1. 段階的処理
データを一括で結論に導かない。  
前処理 → 抽出 → 確認 → 集計 → 可視化  
という工程を分離し、各段階で確認できる構造とする。

### 2. ブラックボックス化しない
ロジックは公開・明示。  
処理順は番号で管理し、再現性を確保する。

### 3. 可視化を前提とする
数値だけで判断しない。  
地図表示・ヒートマップ・集計表など、  
視覚的に確認できることを前提とする。

---

## このツール群で実現すること

- 巨大なプローブデータの段階的整理
- 必要なデータのみの抽出
- 交差点・路線・ゾーン単位での分析
- 流入・流出・OD・経路の可視化
- 分析過程の再現性確保

---

## ソフト群一覧

本ツール群は、用途ごとに複数のソフトで構成されています。  
主な構成は次のとおりです。

### 01　第1スクリーニング
- 第1スクリーニングUI
- 運行ID単位の分割・時系列並べ替え処理

### 02　時間帯存在トリップ集計＋ピーク30分OD抽出
- 指定日の30分帯ごとの存在トリップ集計
- ピーク30分帯の抽出
- 指定30分帯内の最初の点をO、最後の点をDとしたOD抽出 

### 03　推定拠点ゾーン対応
- 運行IDごとの推定拠点ゾーン対応表作成

### 05　トリップビューアー
- 抽出済みトリップの地図確認
- 個別トリップの挙動確認

### 11　交差点ファイル作成ツール
- 交差点中心と枝方向の定義
- 交差点CSV・JPGの作成  
- PyQtの管理UIとHTML地図画面で構成 

### 12　ポリゴンビルダー
- ゾーニング用ポリゴンの作成

### 21　第2スクリーニング
- 指定交差点を通過するトリップの抽出
- 第2スクリーニング済みCSVの作成

### 31　交差点パフォーマンス分析
- 流入枝番・流出枝番の判定
- 所要時間・遅れ時間の算出
- performance.csv の作成

### 32　交差点レポート作成
- performance.csv をもとにした Excel レポート自動生成

### 33　IN-OUT枝判定ビューアー
- 交差点パフォーマンス分析結果の妥当性確認
- 1トリップごとの枝判定結果を地図上で可視化

## 導入方法とフォルダ準備

本ツールは、一般的なソフトウェアのような**インストール作業は不要**です。  
必要なフォルダを所定の場所に配置し、バッチファイルから起動して利用します。

### ①「_EtcAnalyzer」フォルダ

ダウンロードしたZIPファイルには、分析ツール一式をまとめたフォルダ **「_EtcAnalyzer」** が格納されています。  
まず、このフォルダを **Cドライブ直下** または **任意のドライブ直下** に解凍してください。

例

C:\_EtcAnalyzer

#### bat
このフォルダには、プログラムを起動するための**バッチファイル**が格納されています。  
各ツールを起動する場合は、この中の `〇〇〇〇.bat` というファイルをダブルクリックしてください。

#### runtime
このフォルダには、Pythonプログラムを動かすために必要な実行環境が格納されています。  
**この中は触らないでください。**

#### src
このフォルダには、Pythonで記述されたプログラム本体が格納されています。  
**この中も触らないでください。**

---

### ②「_EtcData」フォルダ

次に、分析作業用のフォルダとして **「_EtcData」** というフォルダを**手動で新規作成**します。  
このフォルダには、解析時に中間データや結果データが大量に作成されます。  
そのため、**内蔵SSDなど、高速で十分な空き容量がある場所**に作成してください。

例

C:\_EtcData

目安として、**500GB程度の空き容量**がある場所を推奨します。  
さらに、あらかじめこの中に、必要に応じた作業用フォルダを**手動で作成**しておくと、分析を整理しやすくなります。

---

### ③ ETC2.0プローブデータの生データ

ETC2.0プローブデータ（様式1-2）の生データは、  
**必ずしも `_EtcData` フォルダ内に置く必要はありません。**

生データは非常に大容量となるため、むやみに移動させるのではなく、  
**外付けSSDや外付けHDDに保存したまま利用することも可能**です。

---

### 準備完了

以上で、分析ツールを利用するための準備は完了です。

通常のソフトウェアのように、特別なインストール作業は不要です。  
必要なフォルダを配置し、各バッチファイルから起動して利用してください。
